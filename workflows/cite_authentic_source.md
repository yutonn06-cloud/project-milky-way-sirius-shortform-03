# Workflow: 出典の付与（authentic-source citation）

各SNSサブスキル（`.claude/skills/sns-rewrite/SKILL.md` / `.claude/skills/sns-rewrite-x/SKILL.md`）が呼び出す共通SOP。リライト本文に**実在の権威ソースからの出典1件**を付与する。哲学者引用とは別要素として共存させる。

## 対象ソース

媒体一覧の正本は [reference/authentic-sources.md](../reference/authentic-sources.md)。本SOPはその選定・組み込み方を定義する。

アクティブ・フィード（毎回フェッチして引用候補にする5本）と、長期文脈リファレンス（毎回はフェッチしない2件）の区別はリファレンス側に集約してある。新しい媒体を追加するときは reference/ を更新し、必要に応じて [tools/fetch-sources.ps1](../tools/fetch-sources.ps1) のフィード配列にも追記する。

## 入力

- 選択済みの原文（タイトル・本文）
- 選択済みのターゲット・型／文体

## 取得経路

| 環境 | 経路 |
|---|---|
| ローカル（x-writer 手動実行など） | `pwsh -File tools/fetch-sources.ps1` を先に実行し、`.tmp/sources_today.json` を読む（6時間キャッシュ） |
| クラウドルーチン（短尺動画・cron） | 4本のRSS URLを WebFetch で並列取得し、各フィード上位3件をその場で抽出 |

両経路ともに **15候補（5媒体 × 3件）** を作る。

## 候補の正規化（共通フォーマット）

```json
[
  { "source": "NHK", "title": "...", "url": "https://...", "pubDate": "2026-05-01T...", "summary": "..." },
  ...
]
```

## 関連性の選定ルール（A案・B案で1件ずつ）

1. 原文のテーマ語（タイトル＋本文の名詞）と各候補の `title` + `summary` をキーワード一致で比較。最も重なりの多い候補を**A案用**に採用。
2. **A案と異なる媒体**から、次に重なりの多い候補を**B案用**に採用。
3. キーワード一致が0件の場合は、「人生・働き方・社会・健康・将来」など読者層に親和的なテーマを優先し、最新の候補を採る。
4. それでも候補が選べない場合は、各フィードの**最新1件**から媒体を変えて2件選ぶ（フォールバック）。

## 本文への組み込みフォーマット

リライト本文の **静かなCTAの直前**に、独立した1段落として挿入する。**URLは本文に入れない**（媒体名・記事タイトルのみ）。

```
出典：<媒体名>『<記事タイトル>』
```

例：
```
出典：NHKニュース『2026年版 高齢者世帯調査』
```

- A案・B案で**異なる媒体・異なる記事**を採用すること。
- 文字数は本文の合計（X: 400〜500字 / ショート動画: 300〜400字）に**含める**。
- ショート動画では出典行を可能な限り短くする（〜30字目安、媒体名＋短縮タイトル）。

## Notion への格納

| プロパティ | 値 |
|---|---|
| `出典URL_A` | A案で引用した記事の完全なURL |
| `出典URL_B` | B案で引用した記事の完全なURL |
| `出典媒体` | `"<A案媒体名> / <B案媒体名>"`（例：`"NHK / 厚労省"`） |

## エラーハンドリング

- 4本すべてが取得失敗 → Notion に `ステータス = "失敗"` を書き込んで終了（`メモ` に "出典取得不能" を記録）。
- 1〜2本失敗・3本以上成功 → 残った候補から関連性ルールで2件選び、本フローを継続。
- WebFetch が部分的にタイムアウトした場合は取得済み分で続行。
