# Workflow: 動画パイプライン運用 — 投稿/使用済み/アーカイブの紐づけ

> 2026-06-06 実装（C+A）。「使用済み素材が次バッチまで staging に居座り、どれを使ったか
> 忘れる」ギャップを、即時アーカイブ＋精密アーカイブで解消した。

## 背景（解消した問題）
- 出力素材 ＝ `auto_<原文番号>_<timestamp>_字幕なし.mp4`（Filmora取込素材・[tools/make-video.ps1:820](../tools/make-video.ps1#L820)）。
- 旧 `Invoke-ArchivePosted` は **原文番号glob `auto_<原文番号>_*`** でファイルを拾っていた。
  原文番号は重複する（同じ原典を複数回リライト＝複数レコード）ため、1レコードを使用済みにすると
  **同一原文番号の別レコードの未投稿素材まで巻き込む**クロス汚染があった。
- さらにアーカイブ退避は `-Auto` バッチの先頭でしか走らず、投稿〜次バッチの間、使用済み素材が
  字幕なし\ に残り「どれを既に使ったか」が分からなくなっていた。

## 実装した解消策
### A — 精密アーカイブ（[Invoke-ArchivePosted](../tools/make-video.ps1#L233)）
使用済みレコードに保存された**正確な ファイル名から render stem（`auto_<num>_<stamp>`）を導出**し、
その素材だけを退避する。stamp は1レンダーごとに一意なので、同一原文番号の別レコードは巻き込まない。
ファイル名が空の旧/Xレコードのみ従来の原文番号globにフォールバック（後方互換）。

### C — 即時アーカイブ `-ArchiveOnly`（[make-video.ps1](../tools/make-video.ps1)）
```
powershell -ExecutionPolicy Bypass -File tools\make-video.ps1 -ArchiveOnly
```
生成は一切せず、使用済みレコードの素材だけを 投稿済\ へ即退避して終了する。
**投稿してNotionを使用済みにした直後に1回叩く**運用で、次バッチを待たずにギャップを閉じる。

## 大原則
**投稿状態の唯一の真実源は Notion（ステータス＋ファイル名列）。フォルダは staging。**

## 運用ルール
1. 投稿 → Notion を **`使用済み`** に → **`-ArchiveOnly` を1回実行**。これで素材が即 投稿済\ へ。
2. 「どれが投稿済みか」は Notion を見る：`動画化済`ビュー（作ったが未投稿）／`使用済み`。
   ファイル名列から実ファイルへ辿れる。フォルダ名で推測しない。
3. 再ネタ選定時は**原典マスターの「最新ステータス」**を見て、in-flight（動画化済で未投稿）の
   原文番号を重複ドラフトしない。→ [[genten_master_db]] と連動。
4. `-Auto` バッチも先頭で同じ精密アーカイブを行うので、使用済みにし忘れても次バッチで回収される
   （ただし即時性のため 1 の `-ArchiveOnly` 推奨）。
