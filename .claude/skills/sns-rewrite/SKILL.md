# ショート動画リライトスキル

ショート動画スクリプト**専用**。原文をもとにA案・B案の2本を生成しNotionに書き込む。
他プラットフォーム（Threads / X / ブログ / YouTube台本 / Reels）の最適化は**行わない**。

## 固定仕様（変更不可）

| 項目 | 値 |
|---|---|
| 投稿先 | ショート動画（固定） |
| 文字数 | 300〜400字（A案・B案ともに厳守） |
| 文体 | 敬体（です・ます調）厳守 |
| 出力数 | A案・B案の2本 |
| 差別化軸 | A案とB案で文体・冒頭・構成を変える |

## ワークフロー（順に実行）

### 1. 原文を1本選ぶ

ユーザーが原文を提供した場合はそれを使う。提供されていない場合：

- `skill/references/` 内の45ファイル（ch01〜ch10, s01〜s15, t01〜t10, v01〜v10）から一様ランダムに1ファイル選ぶ
- そのファイル内の10本から一様ランダムに1本選ぶ
- 直近7日以内にNotion DB `087eff43-caa5-41ff-944e-7982f68faef8` に登録済みの「原文番号」と被ったら引き直す（重複チェックは下記 §「Notion 重複チェック」参照）

### 2. ターゲットを1個選ぶ（軸A）

以下10個から一様ランダムに1個。

1. 40代〜50代で、生き方に違和感を感じ始めた女性
2. 「自分の人生これでいいのか」と思い始めた30代〜40代の会社員
3. 周りに合わせて生きてきたけど、もう限界を感じている人
4. スピリチュアルに興味はあるけど、周りには言えずにいる女性
5. 子育てがひと段落して「自分の人生」を取り戻したいと感じている主婦
6. 職場の人間関係に疲れて、ひとりで働く生き方に憧れている人
7. 「自分は普通じゃない」と感じて、ずっと孤独を抱えてきた人
8. 離婚・退職・病気など、人生の転機を経験して価値観が変わった人
9. SNSで自己啓発やスピリチュアルの発信を見て、自分も変わりたいと思い始めた人
10. 組織に属さず、自分の力で生きていきたいと考えている40代〜50代

### 3. A案・B案の文体を選ぶ（軸B）

以下5個から、A案・B案で**異なる2個**を一様ランダムに選ぶ。両方とも敬体維持。

1. やさしく語りかけるけど、核心はズバッと言い切るトーン
2. 静かだけど力強い、ひとり語りのようなトーン
3. です・ます調ベースに、体言止めや問いかけを混ぜてリズムを出す
4. 読者の痛みにまず共感してから、視点を反転させる構成
5. 内省的で詩的、間を活かした語り口

### 4. リライト実行（ショート動画スクリプトとしてA案・B案を生成）

両案ともに以下を厳守する：

- **文字数 300〜400字**（範囲外は不可）
- **冒頭1〜2文に強いフック**（最初の3秒で惹きつける一言）
- **60秒以内で読み切れる量**
- **感情の起伏を意識した構成**（共感→転換→締め）
- **敬体厳守**：語尾は「〜です／〜ます／〜でしょうか／〜ません」のみ
- **タメ語・カジュアル禁止**：「ねえ」「でしょ」「だよね」「じゃん」「だよ」「だね」「〜よね」「〜なの？」などは使用不可
- 体言止め・問いかけ・共感→反転による差別化はOK。ただし語尾は敬体に揃える

A案とB案の差分：
- 同じ原文・同じターゲット
- 3で選んだ**別々の文体**を使う
- 冒頭・構成・締めを明確に変える

### 4.5 出典の付与（authentic-source citation・必須）

[workflows/cite_authentic_source.md](../../../workflows/cite_authentic_source.md) を参照し、4本のRSS（内閣府／厚労省／Yahoo社会／NHK）から本文に組み込む出典をA案・B案で1件ずつ選ぶ。媒体一覧は [reference/authentic-sources.md](../../../reference/authentic-sources.md)。

**取得経路（環境別）：**
- ローカル実行：直前に `pwsh -File tools/fetch-sources.ps1` を実行し、`.tmp/sources_today.json` から候補15件を読み込む（6時間キャッシュ）。
- クラウドルーチン（cron）：4本のRSS URLを WebFetch で並列取得し、各フィード上位3件をその場で抽出する。

**選定ルール：** 原文テーマと候補の `title` + `summary` のキーワード重なりが最も大きいものをA案用に。A案と**異なる媒体**から、次に重なりの大きいものをB案用に。A案・B案で異なる媒体・異なる記事にすること。

**本文への組み込み：**
- 締め（最終文）の直前に、独立した1行として挿入する。
- フォーマット：`出典：<媒体名>『<記事タイトル>』`（**URLは本文に入れない**）。
- ショート動画は文字数枠が狭いため、出典行は **概ね20〜30字以内**（媒体名＋短縮タイトル）に収める。
- 文字数は300〜400字の合計に**含める**。

**Notion格納：**
- `出典URL_A` ← A案で引用した記事の完全URL
- `出典URL_B` ← B案で引用した記事の完全URL
- `出典媒体` ← `"<A案媒体名> / <B案媒体名>"`（例：`"NHK / 厚労省"`）

**音声読み上げ：** 出典行は audio-filler によって自然に読み上げられる（媒体名と記事タイトルのみのため）。URL文字列は本文に含まないので、TTS用の前処理は不要。

**エラーハンドリング：** 4本すべて取得失敗 → `ステータス = "失敗"`、`メモ` に "出典取得不能" を残して終了。

### 5. Notion書き込み（REST API・MCPは使わない）

データソース `087eff43-caa5-41ff-944e-7982f68faef8` に1ページ追加する。

#### ローカル実行（推奨・Windows PowerShell 5.1）

JSON はファイル経由で渡す（Windowsシェルの引用符問題を回避）：

```cmd
:: 1) JSON を一時ファイルに書く（UTF-8、BOMなし）
notepad .tmp\props.json   :: 中身は下のフラットJSONを貼る

:: 2) 呼び出し
powershell -ExecutionPolicy Bypass -File tools\notion-create-page.ps1 -PropertiesFile .tmp\props.json
```

`-PropertiesJson '<flat-json>'`（インライン）は bash / pwsh 7+ で動くが、Windows PowerShell 5.1 経由では内側のダブルクォートが剥がれるため非推奨。

JSONはフラットなプロパティマップ：

```json
{
  "タイトル": "週末が怖い",
  "リライト本文": "（A案の本文）",
  "リライト本文B": "（B案の本文）",
  "投稿先": "ショート動画",
  "ターゲット": "40代〜50代で、生き方に違和感を感じ始めた女性",
  "文体": "A: やさしく語りかける / B: 共感→反転の構成",
  "原文番号": "原文001",
  "文字数": 360,
  "文字数B": "342",
  "採用": "未判定",
  "ステータス": "未使用",
  "出典URL_A": "https://www3.nhk.or.jp/news/...",
  "出典URL_B": "https://www.mhlw.go.jp/stf/...",
  "出典媒体": "NHK / 厚労省"
}
```

スクリプトが Notion REST スキーマ（typed JSON）に変換し、`POST https://api.notion.com/v1/pages` する。標準出力に作成済みページのURLが返る。

#### クラウド実行（直接 REST）

`tools/*.ps1` が動かない環境（claude.ai cloud routine など）では HTTP POST を直接組み立てる：

- `POST https://api.notion.com/v1/pages`
- Headers: `Authorization: Bearer ${NOTION_TOKEN}`, `Notion-Version: 2022-06-28`, `Content-Type: application/json`
- Body（typed JSON）：

```json
{
  "parent": {"database_id": "087eff43-caa5-41ff-944e-7982f68faef8"},
  "properties": {
    "タイトル": {"title": [{"text": {"content": "週末が怖い"}}]},
    "リライト本文": {"rich_text": [{"text": {"content": "（A案の本文）"}}]},
    "リライト本文B": {"rich_text": [{"text": {"content": "（B案の本文）"}}]},
    "投稿先": {"select": {"name": "ショート動画"}},
    "ターゲット": {"select": {"name": "40代〜50代で、生き方に違和感を感じ始めた女性"}},
    "文体": {"rich_text": [{"text": {"content": "A: やさしく語りかける / B: 共感→反転の構成"}}]},
    "原文番号": {"rich_text": [{"text": {"content": "原文001"}}]},
    "文字数": {"number": 360},
    "文字数B": {"rich_text": [{"text": {"content": "342"}}]},
    "採用": {"select": {"name": "未判定"}},
    "ステータス": {"select": {"name": "未使用"}},
    "出典URL_A": {"url": "https://www3.nhk.or.jp/news/..."},
    "出典URL_B": {"url": "https://www.mhlw.go.jp/stf/..."},
    "出典媒体": {"rich_text": [{"text": {"content": "NHK / 厚労省"}}]}
  }
}
```

**音声URL_A・音声URL_Bは書き込まない**。GitHub Actions の `.github/workflows/audio-filler.yml` が06:15 JSTに後から埋める。

#### Notion 重複チェック（直近7日の原文番号）

ステップ1で必要。REST 直叩き：

- `POST https://api.notion.com/v1/databases/087eff43-caa5-41ff-944e-7982f68faef8/query`
- Body:

```json
{
  "filter": {
    "and": [
      {"property": "投稿先", "select": {"equals": "ショート動画"}},
      {"timestamp": "created_time", "created_time": {"past_week": {}}}
    ]
  },
  "page_size": 100
}
```

レスポンスの `results[].properties.原文番号.rich_text[0].plain_text` を集計し、選んだ原文番号と一致したら別の原文を引き直す。

### 6. 終了

最終サマリー（原文タイトル・文字数A/B・出典媒体A/B・NotionページURL）を1〜2行で出力して終了。
バリエーション提案、追加の問いかけ、音声URLへの言及は**しない**。
