# X（Twitter）長文リライトスキル

> **⏸ 【停止中・休眠（2026-06-04〜）】** 本衛星は**ショート動画リライトに専念**する方針となり、X パイプラインは停止しました（claude.ai routine `SNS Rewrite Daily X` = `enabled:false`）。本 SKILL は**削除せず休眠保持**（将来の復活用に仕様を温存）。復活時は (1) 本バナー削除 (2) `RemoteTrigger update {enabled:true}` で routine 再開 (3) 惑星核 §3/§4・constitution brand_voice・meta/foundation の X 記述を稼働へ戻す。司令官決定 → [[docs/handoffs/2026-06-04-cosmos-sync-handoff.md]]。

X長文投稿**専用**。原文をもとに、3つの「型」のうち異なる2つを使ってA案・B案を生成しNotionに書き込む。
他プラットフォーム（ショート動画 / Threads / ブログ / YouTube台本 / Reels）の最適化は**行わない**。

## 固定仕様（変更不可）

| 項目 | 値 |
|---|---|
| 投稿先 | X（固定） |
| 文字数 | 400〜500字（改行含む。A案・B案ともに厳守） |
| 文体 | 敬体ベース。体言止め・問いかけは可。タメ語禁止 |
| 出力数 | A案・B案の2本 |
| 差別化軸 | A案とB案で「型（A型・統計恐怖／B型・暴露怒り／C型・時間切れ）」を別々にする＋引用元（A案＝現代の専門家・公人／B案＝歴史的人物） |
| 必須要素 | 冒頭ニュースフック1つ＋権威者の引用1つ（A案=現代／B案=歴史）＋焦りを残す締め。本文に `出典：...` の独立段落は**入れない** |
| トーン | 読者の不安・焦り・現状への危機感を喚起。「誰かに話してみる」系の静かなCTAは**禁止** |

## 共通の構成テンプレート（全型共通）

400〜500字を、空行で区切られた **6〜8段落** に展開する。各段落は1〜4文。本文に `出典：...` の独立段落は**入れない**（出典素材は要素1のフックに織り込み、URL・媒体名は Notion フィールドのみに格納）。

1. **冒頭ニュースフック**（1〜3文）— §4.5 で選んだ記事の数字・固有名詞・キーフレーズを直接織り込む。**時間軸フレームは§4.5の6種類からランダムに1つ選ぶ**（直近発表／未来予測「2030年には」／世代対比／過去比較／海外比較／断言フック）。「○月○日」「先日」の直近日付フレームの連発は禁止。読者を一気に「これは自分の話だ」と引き込む。**「出典：...」の形では書かない**（媒体名と記事の核は文中に自然に溶かす）。
2. **危機の現状描写**（2〜4文、リスト的に並列）— 仕事／家計／家族／健康／老後がすでに侵食されている具体描写。「気づいたら手取りは減っていた」「同世代の半数は既に動き始めている」など、**取り残されている感覚**を喚起する。
3. **「型」固有の中核**（A型＝統計恐怖、B型＝暴露怒り、C型＝時間切れ。詳細は §3）— 読者を「自分の現状」に直面させる装置。
4. **逃げ場を塞ぐ反転**（1〜2文）— 「『自分は大丈夫』と思っているうちが、いちばん危ない」「気づいた時には、もう遅い」系。**慰めではなく、安心を奪う一撃**。
5. **権威者の引用**（1〜2文）— A案＝**現代の専門家・公人**（経済学者／医師／官僚／経営者など実在の人物）、B案＝**歴史的人物**（セネカ／マルクス・アウレリウス／ユング／ニーチェ／ソクラテス／老子／ハイデガー など）。日本語で。導入文は「○○は、こう警鐘を鳴らしています。」「○○は、こう言葉を残しています。」。**実在の人物・実在の発言のみ。捏造禁止**。
6. **引用と現状の接続**（1〜2文）— 引用を「だから今、何が危ないのか」に橋渡しする。「○○の警告は、今のあなたに向けられている」系。
7. **焦りを残す締め**（1〜2文）— 「立ち止まっている時間はない」「動けるのは、今だけかもしれない」「気づいた人から、抜け出している」系。**「誰かに話してみる／話してみるだけで」系の静かなCTAは禁止**。命令形・脅迫表現も避け、押し売り感は恐れず焦りを残す。

## ワークフロー（順に実行）

### 1. 原文を1本選ぶ

ユーザーが原文を提供した場合はそれを使う。提供されていない場合：

- `skill/references/` 内の45ファイル（ch01〜ch10, s01〜s15, t01〜t10, v01〜v10）から一様ランダムに1ファイル選ぶ
- そのファイル内の10本から一様ランダムに1本選ぶ
- 直近7日以内にNotion DB `087eff43-caa5-41ff-944e-7982f68faef8` に登録済みの「原文番号」と被ったら引き直す（重複チェックは下記 §「Notion 重複チェック」参照）

### 2. ターゲットを1個選ぶ（軸A）

以下10個から一様ランダムに1個。

1. 40代〜50代で、生き方に違和感を感じ始めた女性
2. 「自分の人生これでいいのか」と思い始めた30代〜40代の会社員
3. 周りに合わせて生きてきたけど、もう限界を感じている人
4. スピリチュアルに興味はあるけど、周りには言えずにいる女性
5. 子育てがひと段落して「自分の人生」を取り戻したいと感じている主婦
6. 職場の人間関係に疲れて、ひとりで働く生き方に憧れている人
7. 「自分は普通じゃない」と感じて、ずっと孤独を抱えてきた人
8. 離婚・退職・病気など、人生の転機を経験して価値観が変わった人
9. SNSで自己啓発やスピリチュアルの発信を見て、自分も変わりたいと思い始めた人
10. 組織に属さず、自分の力で生きていきたいと考えている40代〜50代

### 3. A案・B案の「型」を選ぶ（軸B）

以下3つの型から、A案・B案で**異なる2つ**を選ぶ。**型の選定は、§4.5 で取得したA案・B案それぞれのニュース記事の性質に合わせる**（数字が立つ記事→A型、構造的事実が立つ記事→B型、年齢・期限が立つ記事→C型）。型と引用元は組み合わせる：A案＝**現代の専門家・公人**の引用、B案＝**歴史的人物**の引用。

#### A型・統計恐怖型

冒頭で§4.5の記事に含まれる**具体的な数字・割合・調査結果**を提示する。「○○％が」「○年で○倍に」「○万人が既に」など。読者に「自分はどっち側か」を自問させ、**乗り遅れている側に立たされている**ことを突きつける。中核は「動いた人と、動けなかった人の差は、もう開き始めています」系の比較焦燥。引用は**現代の経済学者・統計学者・官僚・経営者**などが残した実発言の警告（例：日銀総裁、財務省事務次官、著名エコノミストの発言など）。締めは「数字は嘘をつきません。差は、ここから開きます」系。

#### B型・暴露怒り型

**ユーザーが特に評価する流れ：軽微な社会批判 → 読者の感覚への同調 → 選択はあなた次第**。攻撃的・煽動的な暴露ではなく、構造への気づきを共感的に提示するのが本型のコア。

冒頭で§4.5の記事に含まれる**構造的事実**（制度／統計／業界慣行）を、軽く批判的なトーンで提示する。「裏を返せば、いまの○○は——」「公に認められたのは——」のような、**暴露というより気づき**の提示。読者の感覚に「その違和感は錯覚ではない／あなたの責任ではない」と同調することで、怒りと裏切られた感覚を**読者自身の中で**呼び起こさせる。引用は**歴史的人物**が同種の構造（権力／支配／搾取／無関心）について残した実発言（例：マルクス・アウレリウス／セネカ／ニーチェ／ルソー／福沢諭吉 など）。締めは「知らされていなかったのは、あなたの責任ではない。でも、知った今、動かない理由はもうない」系で、**動くかどうかを読者の選択に委ねる**。命令形・脅迫表現は厳禁。

#### C型・時間切れ型

冒頭で§4.5の記事に含まれる**年齢・期限・タイムリミット**（年金／介護／雇用／健康／制度改正）を提示する。「○歳までに動けなかった人の○％は」「あと○年で間に合わなくなる」など、読者に**自分の残り時間**を意識させる。中核は「砂時計はもう傾いている」系の時間圧。引用は**現代の医師・心理学者・経営者・思想家**で「時間の有限性」について実発言した人物（例：日野原重明、養老孟司、稲盛和夫、中野信子 など）。締めは「砂時計は、今この瞬間も落ち続けています」系。

**A案とB案で必ず別の型**を選ぶ。引用元の指定（A案=現代／B案=歴史）と組み合わせて、両案が明確に異なる切り口になる。

### 4. リライト実行（X長文投稿としてA案・B案を生成）

両案ともに以下を厳守する：

- **文字数 400〜500字**（改行含む。範囲外不可）
- **§ 共通の構成テンプレート の7要素**（ニュースフック → 危機の現状描写 → 型固有の中核 → 逃げ場を塞ぐ反転 → 権威者の引用 → 引用と現状の接続 → 焦りを残す締め）を必ず通す
- **段落は6〜8個**、空行で区切る。各段落は1〜4文。冗長表現・接続詞の重複は削る
- **本文に `出典：...` の独立段落を入れない**。ニュースの見出し・数字・固有名詞は要素1のフックに自然に織り込む
- **敬体厳守**：語尾は「〜です／〜ます／〜でしょうか／〜ません」を基調にする
- **体言止め・問いかけは積極的に使う**（リズムを作る）。ただし語尾は敬体に揃える
- **タメ語・カジュアル禁止**：「ねえ」「でしょ」「だよね」「じゃん」「だよ」「だね」「〜よね」「〜なの？」などは使用不可
- **引用は実在の人物・実在の言葉**を使う。捏造禁止。記憶があやふやな場合は別の人物に変更する
- **A案＝現代の専門家・公人**（経済学者／医師／官僚／経営者／思想家など）、**B案＝歴史的人物**（哲学者／歴史的指導者）の引用に分ける
- **トーンは焦り・不安・危機感**。「誰かに〜してみるとき／〜してみるだけで」系の静かなCTAは**禁止**。命令形・脅迫表現も避ける（「今すぐ動け」「逃げろ」など）。押し売り感を恐れず、ただし煽動ではなく**気づかせる**圧をかける
- **慰め系の語彙（「怠けではない」「あなたは弱くない」「立ち止まる時間がなかっただけ」）は禁止**

A案とB案の差分：
- 同じ原文・同じターゲット
- 3で選んだ**別々の型**を使う
- **引用元が異なる**（A案=現代／B案=歴史）
- 冒頭フックで使うニュース記事も §4.5 で別媒体・別記事を選ぶ

### 4.5 出典の付与（authentic-source citation・必須）

> **X専用の差分（重要）：** 共通SOP [workflows/cite_authentic_source.md](../../../workflows/cite_authentic_source.md) の「本文への組み込み（`出典：<媒体名>『<記事タイトル>』` の独立段落）」は **X長文では適用しない**。代わりに、選んだニュース記事の見出し・数字・固有名詞・キーフレーズを **要素1の冒頭ニュースフックに自然な日本語として織り込む**。出典URL・媒体名は Notion フィールドのみに格納する。

4本のRSS（内閣府／厚労省／Yahoo社会／NHK）から、A案・B案でそれぞれ1件ずつ選ぶ。媒体一覧は [reference/authentic-sources.md](../../../reference/authentic-sources.md)。

**取得経路（環境別）：**
- ローカル実行（推奨・x-writer）：直前に `pwsh -File tools/fetch-sources.ps1` を実行し、`.tmp/sources_today.json` から候補15件を読み込む（6時間キャッシュ）。
- クラウドルーチン：4本のRSS URLを WebFetch で並列取得し、各フィード上位3件をその場で抽出する。

**選定ルール：**
- 原文テーマと候補の `title` + `summary` のキーワード重なりが最も大きいものをA案用に（A型・統計恐怖を狙うなら数字が立つ記事を優先）。
- A案と**異なる媒体**から、次に重なりの大きいものをB案用に（B型・暴露怒り／C型・時間切れに合う記事を優先）。
- A案・B案で**異なる媒体・異なる記事**にすること（同一媒体・同一記事の二度押しは不可）。
- 選定不能時のフォールバックは [workflows/cite_authentic_source.md](../../../workflows/cite_authentic_source.md) の関連性ルール参照。

**本文への織り込み（X専用）：**
- 要素1の冒頭ニュースフックで、記事の核（数字・固有名詞・発表元）を **散文として自然に提示**する。
- ❌「出典：厚労省『第92回労働政策審議会勤労者生活分科会中小企業退職金共済部会議事録』」のような **「出典：…」の独立段落は禁止**。

**時間軸フレームのローテーション（重要・毎回ランダムに1つ選ぶ）：**

毎回「○月○日／先日」のような**直近日付の枠組み**に偏らない。記事の鮮度は最新RSSから維持しつつ、提示の時間軸は以下から**A案・B案のたびに別々のフレームをランダムに選ぶ**。

| # | フレーム | 開き方の例 |
|---|---|---|
| 1 | 直近発表（控えめ） | 「ある調査が伝えているのは——」「最近の厚労省の発表では」「先ごろ公表された数字では」（**「○月○日」「先日」の連発は避ける**。同じ枠を月に何度も使わない） |
| 2 | 未来予測 | 「2030年、日本の○○は——」「あと○年で、いまの○○は通用しなくなります」「いまの30代が50代になる頃、○○は——」 |
| 3 | 世代対比 | 「あなたの親世代では○○が当然でした。いまは違います」「20年前の○○と、今の○○を並べてみてください」 |
| 4 | 過去比較 | 「10年前、この数字は○○％でした。いまは——」「あの頃は誰もが○○だった。それが、いつからか——」 |
| 5 | 海外比較 | 「○○ではすでに○○％。日本は——」「世界の主要国の中で、○○なのは日本だけです」 |
| 6 | 断言フック（時間軸抜き） | 「働く人の41％が、と聞いて、自分は違うと言い切れるでしょうか」「○○％という数字を、ただの統計だと思えますか」 |

**選定ルール：**
- A案・B案で**異なるフレーム**を選ぶ。両方ともフレーム1（直近発表）にしない。
- 1ヶ月以内に同じフレームを3回以上使わない（Notion DBで `投稿先=X` のリライト本文を確認し、冒頭表現が偏っていないか目視チェック）。
- 記事の発表日が「数日前」だったとしても、フレーム2〜6を選んだ場合は**日付を明示しない**（「2030年には〜」と書くなら、その記事の発表日は本文に出さない）。

A案・B案ともに必須。文字数は400〜500字の合計に**含める**。

**Notion格納（変更なし）：**
- `出典URL_A` ← A案フックで使った記事の完全URL
- `出典URL_B` ← B案フックで使った記事の完全URL
- `出典媒体` ← `"<A案媒体名> / <B案媒体名>"`（例：`"NHK / 厚労省"`）

**エラーハンドリング：** 4本すべて取得失敗 → `ステータス = "失敗"`、`メモ` に "出典取得不能" を残して終了。

### 5. Notion書き込み（REST API・MCPは使わない）

データソース `087eff43-caa5-41ff-944e-7982f68faef8` に1ページ追加する。

#### ローカル実行（推奨・Windows PowerShell 5.1）

JSON はファイル経由で渡す（Windowsシェルの引用符問題を回避）：

```cmd
:: 1) JSON を一時ファイルに書く（UTF-8、BOMなし）
notepad .tmp\props.json   :: 中身は下のフラットJSONを貼る

:: 2) 呼び出し
powershell -ExecutionPolicy Bypass -File tools\notion-create-page.ps1 -PropertiesFile .tmp\props.json
```

`-PropertiesJson '<flat-json>'`（インライン）も使える。bash / pwsh 7+ では問題なく動くが、Windows PowerShell 5.1 から `powershell.exe -File` 経由で渡すと内側のダブルクォートが剥がれて壊れる。**バッチや自動化からは必ず `-PropertiesFile` を使うこと**。

JSONはフラットなプロパティマップ：

```json
{
  "タイトル": "週末が怖い",
  "リライト本文": "（A案の本文・400〜500字）",
  "リライト本文B": "（B案の本文・400〜500字）",
  "投稿先": "X",
  "ターゲット": "40代〜50代で、生き方に違和感を感じ始めた女性",
  "文体": "A: A型・統計恐怖型 / B: B型・暴露怒り型",
  "原文番号": "原文001",
  "文字数": 462,
  "文字数B": "478",
  "採用": "未判定",
  "ステータス": "未使用",
  "出典URL_A": "https://www3.nhk.or.jp/news/...",
  "出典URL_B": "https://www.mhlw.go.jp/stf/...",
  "出典媒体": "NHK / 厚労省"
}
```

スクリプトが Notion REST スキーマ（typed JSON）に変換し、`POST https://api.notion.com/v1/pages` する。標準出力に作成済みページのURLが返る。

#### クラウド実行（直接 REST）

`tools/*.ps1` が動かない環境では HTTP POST を直接組み立てる：

- `POST https://api.notion.com/v1/pages`
- Headers: `Authorization: Bearer ${NOTION_TOKEN}`, `Notion-Version: 2022-06-28`, `Content-Type: application/json`
- Body（typed JSON）：

```json
{
  "parent": {"database_id": "087eff43-caa5-41ff-944e-7982f68faef8"},
  "properties": {
    "タイトル": {"title": [{"text": {"content": "週末が怖い"}}]},
    "リライト本文": {"rich_text": [{"text": {"content": "（A案の本文）"}}]},
    "リライト本文B": {"rich_text": [{"text": {"content": "（B案の本文）"}}]},
```
